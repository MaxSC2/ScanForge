# ScanForge — Инструкции

## Запуск

```bash
# Termux → proot Ubuntu
/sdcard/launch-sf.sh

# Внутри opencode (если вылетел):
cd /sdcard/projects/ScanForge && npm run dev
```

## Интерфейс

### Главное окно
- **Слева** — список страниц проекта
- **Центр** — редактор с областями (регионами)
- **Справа** — инспектор свойств выделенного региона
- **Снизу** — панель статуса (прогресс задач)

### Панель инструментов (сверху)
| Кнопка | Что делает |
|--------|-----------|
| 📄 Загрузить | Изображения (PNG/JPG/WebP) или PDF (каждая страница отдельно) |
| 🎯 Регион | Создать прямоугольную область на странице |
| 🔄 Пайплайн | OCR → Перевод → Экспорт (полный цикл) |
| ▶️ Рендер PNG | Экспорт текущей страницы с переводом |
| ⚙️ Настройки | Языки, OCR, перевод, inpainting, стили текста |

### Клавиши
| Сочетание | Действие |
|-----------|---------|
| `Ctrl+Shift+P` | Открыть диалог пайплайна |
| `Ctrl+Z` | Отмена |
| `Ctrl+Shift+Z` | Повтор |
| `Delete/Backspace` | Удалить выделенные регионы |
| `Ctrl+A` | Выделить все регионы |

## Пайплайн обработки

### Режимы
1. **OCR → Перевод** — только распознать текст и перевести
2. **Полный пайплайн** — OCR → Перевод → Inpainting (очистка) → Экспорт PNG

### Как запустить
1. Загрузи страницы (изображения или PDF)
2. Нажми кнопку пайплайна (иконка Workflow) или `Ctrl+Shift+P`
3. Выбери режим
4. Нажми «Запустить пайплайн»

### Что происходит
1. **OCR** — каждая страница → распознавание текста на выбранном движке → сохранение в регионы
2. **Перевод** — каждый регион с текстом → перевод через выбранный провайдер
3. **Inpainting** — удаление оригинального текста из изображения
4. **Экспорт** — сохранение PNG с наложенным переводом

## OCR (Распознавание текста)

### Доступные движки
| Движок | Где работает | Языки |
|--------|-------------|-------|
| **Превью OCR** | Везде (в браузере) | Любой (имитация) |
| **Windows OCR** | Только Windows | Зависит от системы |
| **Tesseract** | Linux/Windows (CLI) | JP/KO/ZH/EN/RU |
| **PaddleOCR** | Где есть Python | JP/KO/ZH/EN |
| **Manga OCR** | Где есть Python | JP (манга) |

### Установка Python-движков
```bash
# PaddleOCR
pip install paddleocr

# Manga OCR
pip install manga-ocr
```

### Выбор в настройках
`⚙️ Настройки → OCR` — выбери нужный движок.
Для Python-движков покажется напоминание про установку.

## Перевод

### Доступные провайдеры
| Провайдер | Тип | Нужен интернет? |
|-----------|-----|----------------|
| **Локальный черновик** | Имитация | Нет |
| **Офлайн-перевод** (kuromoji + словарь) | Локальный JP→RU/EN | Нет |
| **DeepL API** | Внешний API | Да (бесплатно 500k символов/мес) |
| **LibreTranslate** | Self-hosted API | Нет (свой сервер) |
| **Ollama** | Локальная LLM | Нет (своя модель) |

### Настройка DeepL
1. Получи ключ на https://www.deepl.com/pro-api
2. Вставь в поле API Key в настройках

### Настройка Ollama
```bash
# Установка Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Запуск сервера
ollama serve

# Загрузка модели для перевода
ollama pull llama3.2:1b
# Или для JP→RU: qwen2.5:1.5b  (лучше для японского)
ollama pull qwen2.5:1.5b
```
В настройках укажи эндпоинт (`http://localhost:11434`) и модель.

### SakuraLLM (JP→RU/EN, манга)
**SakuraLLM** — это fine-tuned модели для перевода манги с японского на русский/английский.

```bash
# Установка Ollama (если ещё нет)
curl -fsSL https://ollama.com/install.sh | sh

# Загрузка SakuraLLM (выбери одну)
ollama pull sakura-1.5b    # Лёгкая, 1.5B параметров
ollama pull sakura-7b      # Средняя (рекомендуется)
ollama pull sakura-13b     # Тяжёлая, лучшая точность

# Запуск
ollama serve
```

В настройках ScanForge:
1. В разделе «Перевод» выбери **SakuraLLM (JP→RU/EN, манга)**
2. Эндпоинт: `http://localhost:11434`
3. Модель: `sakura-1.5b` (или та, что загрузил)

Системный промпт автоматически настроен под перевод манги (сохраняет имена,
звукоподражания, культурные термины). При недоступности SakuraLLM падает
на Ollama, затем на локальный черновик.

## Inpainting (очистка фона)

Удаляет оригинальный текст из изображения перед наложением перевода.

### Провайдеры

#### Базовый (Canvas 2D) — по умолчанию
- Работает в браузере, ничего не нужно устанавливать
- Алгоритм: заполняет область текста средним цветом границ + размытие
- Хорошо на манге (однотонные/растровые фоны)
- Плохо на фотографиях и сложных фонах

#### LaMA (IOPaint) — ML модель
- Использует нейросеть LaMA для интеллектуальной очистки
- Требует запущенный IOPaint сервер

### Установка и запуск IOPaint

```bash
# Установка
pip install iopaint

# Запуск с LaMA моделью (CPU)
iopaint run --model=lama --device=cpu --port=8080

# Если есть NVIDIA GPU
iopaint run --model=lama --device=cuda --port=8080
```

После запуска:
1. Открой настройки ScanForge (`⚙️`)
2. В разделе Inpainting выбери «LaMA (IOPaint)»
3. Убедись что эндпоинт `http://localhost:8080`

### Как отправляются данные в IOPaint
1. Оригинальное изображение
2. Маска (белые прямоугольники на чёрном фоне — область каждого региона)
3. IOPaint возвращает очищенное изображение

## Регионы (Bubble Regions)

Регионы — это прямоугольные области на странице, где AI нашёл текст.

### Свойства региона
- **Исходный текст** — что распознал OCR (можно вручную исправить)
- **Перевод** — переведённый текст (можно вручную редактировать)
- **Порядок** — номер для сортировки (важно для чтения)
- **Тип** — речь / мысль / звук и т.д.
- **Видимость** — включить/выключить регион
- **Lock** — заблокировать от изменений
- **Стиль** — шрифт, размер, цвет, обводка, выравнивание

### Ручное редактирование
- Двойной клик по региону → редактирование перевода WYSIWYG
- По умолчанию при загрузке страницы OCR уже запущен
- Регионы можно перемещать и менять размер
- Выделить несколько → групповые операции

## Стили текста

Управляются в настройках проекта (`⚙️ → Текстовые стили`).

### Параметры стиля
- **Шрифт** — семейство (Google Fonts загружаются автоматически)
- **Размер** — в пикселях
- **Цвет заливки** — fill
- **Цвет обводки** — stroke
- **Толщина обводки** — strokeWidth
- **Выравнивание** — left/center/right
- **Межстрочный интервал** — lineHeight

### Шрифты
- Системные шрифты Android доступны без загрузки
- Google Fonts (Noto Sans JP, Roboto, Inter и ~20 других) загружаются автоматически при выборе
- Если шрифт не найден — fallback на системный

## AI-Детекция (One-click)

**Не требует ручного рисования регионов.**

1. Открой изображения (или PDF/CBZ/CBR)
2. Нажми **Пайплайн** в тулбаре (Ctrl+Shift+P)
3. Выбери **"AI-детекция → Перевод → Экспорт"**
4. Кнопка запуска

**Что произойдёт:**
- PaddleOCR просканирует каждую страницу целиком
- Найдёт все текстовые блоки и распознает текст
- Автоматически создаст регионы (sourceText + координаты)
- Запустит перевод (по настройкам) и экспорт с инпайтингом

**Требования:** `pip install paddleocr paddlepaddle`. Если не установлен — fallback на EasyOCR.

**После авто-детекции** можно вручную поправить регионы, если AI что-то пропустил.

## API-сервер (обработка на телефоне)

Отдельный FastAPI сервер, который принимает главу и возвращает готовый ZIP с переведёнными страницами.

### Запуск сервера

```bash
cd /sdcard/projects/ScanForge/server
pip install -r requirements.txt
python main.py
```

Сервер будет доступен на `http://IP-телефона:8765`.

### Отправка из приложения

1. Нажми **Сервер** в тулбаре (иконка базы данных)
2. Введи хост и порт (если сервер на том же устройстве — `localhost:8765`)
3. Включи флажок "Сервер включён"
4. Нажми **Отправить**
5. Прогресс отображается через WebSocket
6. Когда готово — **Скачать ZIP**

### Настройка на телефоне

Сервер работает в Termux. Убедись, что:
- FastAPI запущен (`python main.py`)
- Порт доступен (не заблокирован файрволом)
- Если отправляешь с другого устройства — вместо `localhost` укажи IP телефона

## Мониторинг новых глав (Источники)

Позволяет подписаться на мангу и автоматически загружать новые главы.

### Как работает

1. Нажми **Источники** (иконка робота) в тулбаре
2. Добавь источник:
   - **RSS** — ссылка на RSS фид сайта с главами
   - **Scrape** — URL страницы со списком глав (парсит ссылки)
3. Нажми **Check** для проверки или **Check all** для всех сразу
4. Перейди на вкладку **Главы** — новые появятся там
5. Нажми **Get** для загрузки страниц главы и импорта в проект

### После импорта

Загруженные страницы появятся в проекте. Можно запустить обычный пайплайн (OCR → Перевод → Экспорт) или AI-детекцию.

### Советы

- RSS работает надёжнее scrape (стандартный формат)
- Для scrape нужно, чтобы ссылки на главы содержали слово "chapter" или "глава"
- Загруженные главы отмечаются ✓ и не загружаются повторно

## Если что-то не работает

### Пайплайн не запускается
1. Сохрани проект (автосохранение должно сработать)
2. Проверь что есть хотя бы одна страница
3. Проверь настройки (язык, OCR движок, провайдер перевода)

### IOPaint не подключается
1. Запущен ли сервер? `curl http://localhost:8080/`
2. Правильный ли порт в настройках?
3. Нет ли CORS ошибок? (IOPaint по умолчанию разрешает все origins)

### OCR не работает (Python)
- Установлен ли Python пакет? (см. инструкции выше)
- Доступен ли Python из командной строки?
- Попробуй переключиться на «Превью OCR» для теста

### Перевод пустой
- Выбран ли правильный провайдер?
- Для DeepL/Libre — правильный ли API ключ/URL?
- Для Ollama — запущен ли сервер и загружена ли модель?
