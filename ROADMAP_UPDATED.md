# ScanForge — Обновлённый план развития

## Философия проекта

> ИИ делает черновую работу быстро. Профессионал доводит результат до высокого качества внутри одной удобной программы.

ScanForge — гибридный инструмент:
1. Запустить ИИ-пайплайн
2. Быстро найти и исправить все проблемные места
3. Сделать финальную вёрстку и очистку
4. Всё это — не выходя из одной программы

## Позиционирование

| Инструмент | Подход | Сильная сторона | Слабая сторона | Для кого |
|---|---|---|---|---|
| Ballons Translator | Сильная автоматизация | Быстрый один клик | Средний ручной контроль | Быстрый перевод |
| Браузерные расширения | Максимальная скорость | Удобно читать | Почти нет контроля | Чтение |
| **ScanForge** | **Гибрид + сильный ручной контроль** | Профи может довести до идеала в одной программе | Чуть медленнее "одного клика" | **Качественный перевод + контроль** |

Ключевая идея: *"ИИ помогает, но я остаюсь хозяином результата и могу всё поправить удобно и быстро."*

## Приоритеты

| # | Что делать | Почему важно | Срочность |
|---|---|---|---|
| 1 | **Мощный и удобный Region Editor** | Профи должен быстро и приятно править всё | Очень высокая |
| 2 | Качественный OCR + несколько движков | Основа всего | Очень высокая |
| 3 | Хороший перевод (локальный + API) | Черновик должен быть уже нормальным | Высокая |
| 4 | Инструменты очистки (Inpainting + ручная кисть) | Профи часто правит именно здесь | Высокая |
| 5 | Удобная работа со стилями и typesetting | Финальное качество | Высокая |
| 6 | Job system + умные политики | Чтобы не приходилось всё переделывать вручную | Средняя |

## Phase 1: Базовый гибридный workflow (2–3 месяца)

**Цель:** Человек запускает ИИ → получает черновик → может быстро и удобно всё поправить.

### OCR
- [ ] Интеграция PaddleOCR (основной)
- [ ] Поддержка нескольких движков + выбор движка для проекта
- [ ] Сохранение уверенности (confidence) для каждого региона
*(Статус: 5 провайдеров уже есть — WinRT, Tesseract, Paddle, MangaOCR, Preview; UI выбора и fallback chain работают)*

### Перевод
- [ ] Минимум 1–2 рабочих провайдера (локальная модель + опционально API)
- [ ] Возможность перевести один регион / несколько / всю страницу заново
*(Статус: 6 провайдеров уже есть — DeepL, Libre, Ollama, offline kuromoji, local, mock)*

### Region Editor (самое важное в фазе)
- [ ] Удобное выделение и редактирование регионов
- [ ] Изменение формы региона
- [ ] Объединение / разделение регионов
- [ ] Ручное создание региона
- [ ] Быстрое редактирование текста региона
*(Статус: выделение, мульти-селект, группировка, z-index, трансформер, снаппинг — есть. НЕТ: слияние/разделение, изменение формы)*

### Базовая очистка
- [ ] Простой inpainting (удаление оригинального текста)
- [ ] Ручная кисть для локальной очистки
*(Статус: inpainting есть (Canvas 2D border-fill + blur). НЕТ ручной кисти)*

### Экспорт
- [ ] Экспорт с текущим состоянием (текст + стили)
*(Статус: есть — renderPageToBlob с inpaint + typesetting; batch CBZ)*

**Результат Phase 1:** Профи может открыть главу → запустить ИИ → вручную поправить все косяки в одном окне и получить приемлемый результат.

### One-click перевод
- Загрузил файл → нажал кнопку → AI сам:
  1. Распознал текст (OCR)
  2. Определил где баблы (region detection)
  3. Перевёл
  4. Раскидал перевод по баблам
  5. Почистил фон (inpaint)
  6. Наложил текст
  7. Отдал готовую страницу
- Без ручного создания регионов — AI сам понимает структуру страницы
